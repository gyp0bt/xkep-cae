# status-129: CR梁ヘリカル要素の収束問題の根本原因診断

[← README](../../README.md) | [← status-index](status-index.md) | [← status-128](status-128.md)

**日付**: 2026-03-07
**テスト数**: 2271（変更なし: fast 1691 + 1 xfailed / slow 362 - 9 xfailed + deprecated 218）

## 概要

7本撚線NCP曲げ揺動テストの収束失敗（10°OK / 45°以上NG）の根本原因を詳細診断した。
**結論: 問題はNCP接触やアクティブセットではなく、CR梁要素の寄生軸力が根本原因**。

## 診断結果

### 排除された原因

| 候補 | 結果 | 根拠 |
|------|------|------|
| NCP接触定式化 | ❌ 原因ではない | NCP active = 0（接触が全く起きない状態でも失敗） |
| アクティブセットチャタリング | ❌ 原因ではない | 接触ペアが活性化しない |
| 時間積分 / 時間増分管理 | ❌ 原因ではない | 適応時間増分でも改善しない |
| 梁表面平滑化関数 | ❌ 原因ではない | 接触が起きないため無関係 |
| Mortar離散化 | ❌ 原因ではない | 同上 |
| k_pen感度 | ❌ 原因ではない | 同上 |
| メッシュ密度 | ❌ 原因ではない | 16→32要素/ピッチでも改善しない |
| 数値微分eps | ❌ 原因ではない | eps=1e-7は適切（eps感度テストで確認） |
| Gram-Schmidt不連続性 | ❌ 原因ではない | Rodrigues回転に変更しても改善しない |

### 特定された根本原因: CR梁の寄生軸力

**メカニズム**:
1. CR定式化は `d_cr[6] = L_def - L_0`（弦長変化）を軸変形として使用
2. 梁が曲がると弦が伸長: `dL ≈ Δy²/(2L)` （二次効果）
3. 軸剛性 EA/L ≈ 4×10⁸ がこの微小な弦長変化に乗じて**巨大な寄生軸力**を生成
4. 曲げ力 ±3.7 N に対し、寄生軸力 95.7 N（25倍）
5. **EA/(EI/L²) ≈ 9000** の剛性比が問題の本質

**影響**:
- u=0 での接線剛性は軸-曲げ結合が0（二次項の一次微分 = 0）
- 最初のNewtonステップで残差が**1475倍に増大**
- 後続の収束は線形（次数 ~1.1）で、二次収束しない

## 実施した修正と結果

### 1. Rodrigues回転フレーム（残存）

- `_rodrigues_rotation()` 関数を追加
- `timo_beam3d_cr_internal_force` のcorotatedフレーム計算を Gram-Schmidt → Rodrigues に変更
- `_cr_extract_deformations` のRodrigues式のバグ修正（`R_rod @ R_0` → `R_0 @ R_rod.T`）
- **結果**: 理論的に正しい最小回転を使用。直線梁の収束を4反復に改善（従来5反復）。ヘリカル梁の根本問題は解決しない
- **回帰テスト**: 全1691 fast テスト PASS

### 2. d_cr[6] の修正試行（全て撤回）

| 手法 | 概要 | 結果 |
|------|------|------|
| Hermite弦短縮補正 | `d_cr[6] -= L/30*(2θ₁²-θ₁θ₂+2θ₂²)` | 寄生力33%削減だが不十分 |
| 初期軸射影法 | `d_cr[6] = e_x_0 · Δu` | 大回転で不安定（特異行列発生） |
| 弦長＋射影の平均 | `d_cr[6] = (L_def-L_0 + e_x_0·Δu)/2` | 直線梁が収束しなくなる |

**教訓**: d_cr[6] の定義変更はCR定式化の整合性（内力と接線の一貫性）を破壊する。寄生軸力は接線情報としても必要であり、単純に除去できない。

## 今後の修正方針

### 推奨: 解析的接線剛性の実装（優先度: 高）

数値微分接線を解析的接線に置き換える:
- `K_T = B^T @ K_e @ B + K_σ`
- B: 自然変形の全体DOF微分（プロジェクター行列）
- K_σ: 幾何剛性（現在の内力に依存、軸力-曲げ結合を含む）
- **期待効果**: 二次収束の回復、ヘリカル梁でも5-7反復で収束

### 代替案

1. **Updated Lagrangian for beams**: 収束ステップごとに参照配置を更新。回転DOFの取り扱いが課題
2. **適応荷重増分の強化**: ヘリカル梁では初期荷重増分を自動的に小さくする
3. **line searchの調整**: diverge_factor を 3.0 → 1.5 に厳格化
