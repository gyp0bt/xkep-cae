## 実施内容

### 1. 2D投影可視化フレームワーク

従来の mplot3d による3Dプロットはアスペクト比が1:1:1にならず、撚線構造の確認に不適切だった。
四元数モジュール（`xkep_cae/math/quaternion.py`）を活用し、任意視点からの3D→2D投影方式に変更。

| ヘルパー関数 | 内容 |
|------------|------|
| `_project_3d_to_2d()` | 四元数回転で3D座標を2D投影 + 深度計算 |
| `_beam_surface_polys_2d()` | 梁表面メッシュを2D投影ポリゴンに変換 |
| `_draw_wire_2d()` | 深度ソート付き1素線描画 |
| `_draw_wires_2d()` | 全素線を層別色分けで描画 |
| `_setup_2d_projected_ax()` | 等方アスペクト比の軸設定 |

### 2. 新規検証プロット3種

| 関数 | 内容 | 出力 |
|------|------|------|
| `plot_contact_force_vectors_3d()` | 接触力ベクトル場 + ギャップ分布 + 推移 + 断面ビューの4パネル診断 | `contact_force_vectors_3d.png` |
| `plot_deformed_twisted_wire_3d()` | 変形前後を3視点（斜め・端面・側面）で比較 | `deformed_twisted_wire_3d.png` |
| `plot_twisted_wire_3d_surface_multi()` | 19本/37本の初期形状を層別色分け・2視点で表示 | `twisted_wire_3d_surface_multi.png` |

**診断パネルの意義**: 貫入量の空間分布やチャタリングの原因を可視化することで、数値困難の特定を支援する。

## 設計上の懸念・ユーザーへの確認事項

1. **37本Layer1+2の不収束原因**: 276ペアが同時にアクティブ化→残差発散。段階的活性化メカニズム（例: g_on を徐々に縮小）の導入が必要かもしれない
2. **2D投影方式の選択**: ユーザーから「mplot3dよりも2D投影が見やすい」との指摘があり変更した。四元数回転を使う方式は正確な等方投影が可能
3. **接触一斉活性化の抑制**: 現在のNCP実装は全候補ペアを同時評価するため、大規模問題で不安定化しやすい。段階的な活性化戦略（例: Layer別の段階的投入）が有効か検討必要
