## 技術的知見: ヘリカル弦近似による初期貫入

### 発見

ヘリカル撚線のbeam要素は直線セグメント（弦）で近似されるため、**弦の中点がヘリックス弧よりも中心軸に近くなる**。これにより、2D断面上では非接触でも3Dセグメント間で仮想的な初期貫入が発生する。

### 定量分析

弦の中点距離 = `r_lay * cos(θ_seg/2)` where `θ_seg = 2π / n_elems_per_pitch`

| n_elems/pitch | θ_seg | cos(θ/2) | gap=0時の最大貫入 | ワイヤ半径比 |
|:---:|:---:|:---:|:---:|:---:|
| 4 | 90° | 0.707 | 0.586 mm | 58.6% |
| 8 | 45° | 0.924 | 0.152 mm | 15.2% |
| 12 | 30° | 0.966 | 0.068 mm | 6.8% |
| 16 | 22.5° | 0.981 | 0.038 mm | 3.8% |
| 24 | 15° | 0.991 | 0.017 mm | 1.7% |

### 最小ギャップの理論式

初期貫入を防ぐ最小ギャップ: `gap_min = d * (1/cos(θ_seg/2) - 1)`

## 変更詳細

### 1. ContactConfig: adjust_initial_penetration + position_tolerance

```python
@dataclass
class ContactConfig:
    # S3改良12
    adjust_initial_penetration: bool = True   # 初期貫入をgap_offsetで補正
    position_tolerance: float = 0.0           # 節点移動の閾値 [m]
```

- **adjust=True（デフォルト）**: `store_initial_offsets()` で初期貫入量を記録し、`update_geometry()` で `g_eff = g_raw - gap_offset` として補正
- **adjust=False**: 初期貫入があればValueError
- **position_tolerance > 0**: tolerance未満のギャップを持つペアの節点を接触位置に移動

### 2. solver_ncp.py: 初期貫入チェック

ソルバー初期化時に参照座標でペア検出・gap計算を実行:
- 初期貫入があれば adjust=True で自動補正、adjust=False でエラー
- 進捗表示で貫入ペア数・最大貫入量を報告

### 3. メッシュ密度制約

`make_twisted_wire_mesh` で16要素/ピッチ未満の場合に `UserWarning` を発生。

### 4. 19本テスト更新

全テストの `n_elems_per_strand` を 4 → 16 に変更。

## 19本NCP収束チューニング中間報告

### gap=0.0005 + 16要素（接触なし）
- 初期貫入なし、接触ペアがアクティベートされない（g_on=gap=0.0005のため）
- 全ステップ収束するが接触力ゼロ（物理的に不正確）

### gap=0 + 16要素（密着配置）
- 552ペアに初期貫入（最大0.038mm）→ gap_offset補正
- 接触ペアが一斉にアクティベート → 力残差が発散
- k_pen continuation + omega制御でも収束せず

### 根本課題
1. **ペア数の急増**: gap=0で552ペアが同時アクティブ化 → 鞍点系の条件数悪化
2. **k_penの高値**: 16要素でL_avg=0.003m → k_pen=8.4e6（要素が短いため）
3. **段階的アクティベーション**: gap=0では段階的にならない

## 確認事項・設計上の懸念

1. **gap_offsetの大変形時の妥当性**: gap_offsetは初期状態で計算され全ステップに適用される。大変形では弦近似パターンが変化するため正確性が低下する。しかし変形後のペア再検出では新ペアはoffset=0で生成されるため、段階的に影響は軽減される。
2. **理想撚線のgap**: 理想撚線は初期状態で密着（gap=0）が物理的に正しいが、NCP収束の観点では微小gapで段階的アクティベーションが必要。
3. **メッシュ密度のトレードオフ**: 16要素/ピッチは接触精度に必要だが、DOF数が4倍になりソルバー負荷が増大。
