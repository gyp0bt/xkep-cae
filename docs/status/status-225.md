## 概要

NR ループ内ジオメトリ更新で `np.clip(s, 0, 1)` が生む **勾配不連続**（C0 だが C1 でない）を
Huber 風スムースクランプ `_smooth_clip_01` で解消。

既存 499 テスト全通過（回帰なし）。ただし動的三点曲げ（E=25, push=30）は frac=0.60 で

## 変更内容

### `xkep_cae/contact/geometry/_compute.py`

1. **`_smooth_clip_01(s, epsilon=0.02)` 追加**

   ```
   s < -ε       → 0                        (ハードクランプ)
   -ε ≤ s < ε   → (s + ε)² / (4ε)          (C1 二次遷移, 下端)
   ε ≤ s ≤ 1-ε  → s                        (線形通過)
   1-ε < s ≤ 1+ε → 1 - (1+ε - s)² / (4ε)  (C1 二次遷移, 上端)
   s > 1+ε      → 1                        (ハードクランプ)
   ```

   - 最大偏差: ε/4 = 0.005（s=0, s=1 での差）
   - 全境界で値 C0, 勾配 C1 連続

2. **`_closest_point_segments_batch` の非平行分岐 6 箇所を `_smooth_clip_01` に置換**
   - 平行セグメント分岐は `np.clip` のまま（端点遷移問題なし）

### NR 停滞の原因分析

- 残差比 0.9999 で固定 → 接線修正が実質ゼロ
- **残差関数**: smooth_clip で C1 化済み（改善）
- **接線剛性**: ∂s/∂u, ∂t/∂u を含まない → 残差の実勾配と乖離

smooth_clip は残差の連続性改善（正しい方向）だが、接線剛性に
∂gap/∂s × ∂s/∂u の項がないため大変形域で NR ステップ方向が不正。
